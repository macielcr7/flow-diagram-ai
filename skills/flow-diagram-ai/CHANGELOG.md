# Changelog

## 1.2.0

- added explicit execution modes for JSON-only versus ready-project delivery;
- added `scripts/materialize_starter.py` to copy the starter and inject a validated snapshot into a target folder;
- added project bootstrap guidance and a prompt pattern for folder-based deliveries;
- expanded the smoke test to verify starter materialization.

## 1.0.0

- fixed the snapshot contract around the current dashboard schema;
- bundled validated example snapshots for multiple architecture shapes;
- added import checklist, scenario prompt recipes, and prose-to-JSON guide;
- bundled a frontend-only starter that opens directly in the dashboard;
- added a smoke test script for examples and starter integrity.
